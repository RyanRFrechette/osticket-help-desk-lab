# osTicket Help Desk Lab Case Study

## Problem
Entry-level help desk and IT support roles expect technicians to work inside ticketing systems, document user issues clearly, communicate with end users, add internal notes, and close tickets with useful resolution details.

## Goal
Build a realistic osTicket lab that demonstrates practical ticket queue workflow for remote Help Desk, IT Support Specialist, Technical Support, SaaS Support, and Application Support roles.

## Lab Setup
- osTicket running locally in a Docker-based lab environment
- Staff/admin access configured
- Test user/customer created
- Departments and help topics configured
- Six realistic support tickets created, documented, resolved, and captured with screenshots

## Ticket Workflows Completed
1. User Cannot Log In
2. Password Reset Request
3. Shared Folder Access Request
4. Slow Computer / Basic Triage
5. Software Install Request
6. New User Onboarding Request

## What I Did
- Created realistic support tickets from a test end user
- Opened and reviewed tickets in the osTicket staff panel
- Added internal troubleshooting notes
- Wrote customer-facing replies
- Resolved/closed each ticket
- Captured proof screenshots for ticket creation, ticket detail views, internal notes, and closed ticket queues
- Documented each ticket workflow in separate markdown files
- Updated the GitHub README with a full screenshot walkthrough

## Skills Demonstrated
- Ticket intake and categorization
- Help topic usage
- Internal documentation
- Customer communication
- Basic troubleshooting workflow
- Password reset and account access awareness
- Shared folder permissions awareness
- Software install approval/licensing awareness
- New user onboarding documentation
- Git/GitHub portfolio documentation
- Screenshot-based technical proof

## Challenges
The main challenge was keeping the project organized while capturing many screenshots and documenting each ticket consistently. I improved the workflow by batching each ticket: collect clean screenshots, write the ticket documentation, update the README, commit once, and push once.

## Result
The finished lab shows a complete help desk ticketing workflow from intake to closure. A hiring manager can review the repository and see evidence that I can work through tickets professionally, document troubleshooting steps, communicate with users, and keep a support queue organized.

## Next Improvement
The next improvement would be adding a short video walkthrough or expanding the lab with Microsoft 365 / Entra ID user-support scenarios.
