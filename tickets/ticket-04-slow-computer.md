# Ticket 04: Slow Computer / Basic Triage

## Scenario
A test user, Taylor Morgan, reported that their computer was running slowly, apps were taking a long time to open, and switching between programs was delayed.

## User Request
Taylor Morgan submitted a slow computer ticket after restarting once and still experiencing poor performance. The user needed help troubleshooting the issue so they could continue working normally.

## Troubleshooting Steps
1. Confirmed the user's slow computer report.
2. Opened the ticket in the osTicket staff panel.
3. Added an internal note documenting the initial performance triage process.
4. Added a follow-up internal note documenting additional checks such as Task Manager resource usage, startup apps, disk space, pending Windows updates, and recent software changes.
5. Posted a customer-facing reply with basic troubleshooting steps.
6. Resolved the ticket after documenting the support response.

## Resolution
The ticket was handled as a basic slow computer troubleshooting scenario. The support response recommended closing unnecessary applications, checking disk space, restarting again after updates, and continuing troubleshooting if the issue persisted.

## Skills Demonstrated
- Basic performance troubleshooting
- User communication
- Internal troubleshooting notes
- Follow-up documentation
- Ticket status tracking
- Customer-facing response
- Resolution documentation
- Closed ticket queue proof
