# Ticket 06: New User Onboarding Request

## Scenario
A test user, Taylor Morgan, submitted a new user onboarding request for an employee starting soon.

## User Request
Taylor Morgan requested help setting up a new employee account, email, application access, and shared folder permissions.

## Troubleshooting Steps
1. Confirmed the new user onboarding request.
2. Opened the ticket in the osTicket staff panel.
3. Added an internal note documenting the onboarding triage process.
4. Documented likely next checks: employee name, start date, manager, required applications, email needs, and folder/group access.
5. Posted a customer-facing reply explaining the onboarding checklist process.
6. Resolved the ticket after documenting the support response.

## Resolution
The ticket was handled as a common new user onboarding request. The support response documented that IT should confirm the employee details, manager, start date, required applications, email needs, and folder permissions before completing setup.

## Skills Demonstrated
- New user onboarding workflow
- Account setup documentation
- Email and application access planning
- Shared folder permission awareness
- Internal troubleshooting notes
- Customer communication
- Resolution tracking
- Closed ticket queue proof
