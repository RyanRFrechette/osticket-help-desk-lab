# Ticket 02: Password Reset Request

## Scenario
A test user, Taylor Morgan, reported that they forgot their company account password and could not sign in.

## User Request
Taylor Morgan submitted a password reset request because they could not access their company account and needed help regaining access so they could continue working.

## Troubleshooting Steps
1. Confirmed the user's password reset request.
2. Opened the ticket in the osTicket staff panel.
3. Added an internal note documenting the password reset triage process.
4. Documented likely next checks: identity verification, account status, and password reset process.
5. Posted a customer-facing reply explaining the next steps.
6. Resolved the ticket after documenting the support response.

## Resolution
The ticket was handled as a common password reset request. The user was instructed to verify their identity before the reset could be completed. The ticket was documented with an internal triage note, customer-facing response, and closed ticket queue proof.

## Skills Demonstrated
- Password reset workflow
- Account access troubleshooting
- Identity verification awareness
- Ticket documentation
- Internal notes
- Customer communication
- Resolution tracking
- Closed ticket queue proof
