# Ticket 05: Software Install Request

## Scenario
A test user, Taylor Morgan, requested help installing approved work software because they did not have permission to install it themselves.

## User Request
Taylor Morgan submitted a software install request asking IT to help install approved software or explain whether approval was needed.

## Troubleshooting Steps
1. Confirmed the user's software install request.
2. Opened the ticket in the osTicket staff panel.
3. Added an internal note documenting the software install triage process.
4. Documented likely next checks: requested software name, approval status, licensing requirements, and whether IT should complete the install.
5. Posted a customer-facing reply explaining the approval and installation process.
6. Resolved the ticket after documenting the support response.

## Resolution
The ticket was handled as a common software installation request. The support response documented that IT should confirm the software name, verify approval and licensing requirements, and complete the installation or provide next steps once approved.

## Skills Demonstrated
- Software installation request workflow
- Approval and licensing awareness
- User communication
- Internal troubleshooting notes
- Ticket documentation
- Resolution tracking
- Closed ticket queue proof
